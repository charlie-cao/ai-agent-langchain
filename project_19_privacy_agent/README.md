# 🔒 Project 19 — 数据隐私合规 Agent

## 商业价值说明书

### 为什么这个 Agent 有市场需求？
中国《个人信息保护法》（PIPL）2021 年正式实施，GDPR 在欧洲执行力度持续加强。企业面临的核心痛点：
- **PII 泄露风险高**：客服日志、销售邮件、数据库 dump 中随处可见敏感信息
- **合规成本高**：专业数据隐私工程师稀缺，年薪 50-100 万
- **人工审查效率低**：数千份文档的 PII 扫描需要数周
- **违规代价大**：PIPL 最高罚款 1 亿元或年营收 5%

本 Agent 自动化 PII 检测和脱敏，将合规审查时间从数周压缩到数分钟。

## 项目简介

数据隐私合规 AI Agent，支持多类型 PII 检测（手机号/身份证/邮箱/IP/银行卡）、自动脱敏、合规审计日志，专为中文场景优化。

## 技术架构

```
输入: 需要检测的文本
    ↓
pii_detector.py → PII 检测 (正则引擎):
                   手机号: 1[3-9]\d{9}
                   身份证: (?<!\d)\d{15}(?:\d{3})?(?!\d)  ← 中文\b边界修复
                   邮箱: [a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+
                   IP地址: \b\d{1,3}(\.\d{1,3}){3}\b
                   银行卡: (?<!\d)\d{16}(?:\d{3})?(?!\d)
    ↓
                → 脱敏处理:
                   手机: 138****8888
                   身份证: 110***...***1234
                   邮箱: u***@domain.com
    ↓
                → 审计日志 (append-only)
    ↓
agent.py        → 综合调度
    ↓
app.py (Streamlit) ← UI
api.py (FastAPI)   ← REST API
```

## 功能特性

- ✅ 5类 PII 检测（手机/身份证/邮箱/IP/银行卡）
- ✅ 中文边界处理（`(?<!\d)` 替代 `\b` 解决中文字符问题）
- ✅ 分级脱敏策略（不同 PII 类型不同脱敏格式）
- ✅ Append-only 审计日志
- ✅ 批量文本处理支持

## 技术要点

**中文 `\b` 边界问题修复**：Python 正则 `\b` 在汉字与数字相邻时不生效（因为汉字被视为 \w 字符）。身份证号正则从 `\b\d{15}(\d{3})?\b` 改为 `(?<!\d)\d{15}(?:\d{3})?(?!\d)`，确保中文场景下正确匹配。
