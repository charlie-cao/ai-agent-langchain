# 🏥 Project 13 — 医疗健康 Agent

## 商业价值说明书

### 为什么这个 Agent 有市场需求？
中国每年门诊量超 60 亿人次，三甲医院平均等待时间 2-4 小时，"看病难"是长期社会痛点。关键问题：
- **预检分诊不精准**：大量"小病"涌入三甲医院，真正危重患者等候时间被拉长
- **EHR 数据杂乱**：医生每天看 80-100 个病人，手工整理病历浪费大量时间
- **基层医疗能力弱**：社区诊所缺乏培训，难以准确分诊

本 Agent 通过 AI 辅助分诊和 EHR 智能摘要，将医生信息处理效率提升 3-5 倍，同时帮助患者在就诊前了解紧迫程度。

### 市场竞品分析
| 竞品 | 优势 | 劣势 |
|------|------|------|
| 微医/好大夫 | 用户基础大 | 主要是挂号，AI能力弱 |
| 平安健康 | 医生资源多 | 商业化导向，费用高 |
| 医渡云 | 医院IT系统整合 | 企业端为主，不开放 |
| **本项目** | 本地运行，数据不出院，可定制 | 需医疗机构自行部署 |

## 项目简介

医疗健康 AI Agent，集成患者分诊（emergency/urgent/routine）、预约管理（UUID-based CRUD）、电子病历智能摘要（血压/体温/用药/诊断提取）。

## 技术架构

```
患者输入 (症状描述 / 病历文本)
    ↓
symptom_checker.py  → 分诊评级 (emergency/urgent/routine)
                       PII脱敏 (电话/身份证号)
                       严重度评分 (0-10)
    ↓
appointment_tool.py → 预约 CRUD (UUID前8位作ID, 患者ID脱敏)
ehr_summary.py      → 电子病历摘要
                       血压/体温/用药提取 (正则)
                       慢性病/高危标记
    ↓
agent.py            → 分诊路由 + 日志记录
    ↓
app.py (Streamlit)  ← UI
api.py (FastAPI)    ← REST API
```

## 功能特性

- ✅ 三级分诊（紧急/急症/普通）+ 严重度评分
- ✅ PII 自动脱敏（手机号、身份证）
- ✅ 预约全生命周期管理（创建/查询/更新/取消）
- ✅ EHR 结构化摘要（生命体征/用药/诊断）
- ✅ 高危条件自动标记（高血压/糖尿病/心脏病）
