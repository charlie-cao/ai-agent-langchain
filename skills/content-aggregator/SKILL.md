# Content Aggregator

内容聚合与报告生成技能。

## 使用场景

- **每日简报**：汇总当天重要咨讯
- **热点专题**：某一话题的多角度综述
- **周/月报**：周期性信息整合
- **竞品动态**：多竞品信息一网打尽
- **舆情报告**：正负面信息分类呈现

## 输入类型

- 多篇独立文章/新闻
- 多个RSS源当日更新
- 指定话题的搜索结果
- 某个网站的最新内容列表

## 输出类型

| 类型 | 受众 | 篇幅 |
|------|------|------|
| 早报 | 快速浏览 | 500字内 |
| 日报 | 深度阅读 | 1500字内 |
| 周报 | 管理层 | 结构化要点 |
| 专题报告 | 专业读者 | 深度分析 |

## 工作流程

```
1. 收集（30%精力）
   - 调用 content-harvester 批量采集
   - 去重、过滤低质量

2. 阅读（30%精力）
   - 每篇快速过目，提炼核心
   - 标注重要程度

3. 聚合（20%精力）
   - 按维度/主题分类
   - 合并相似内容
   - 找出信息空白

4. 输出（20%精力）
   - 按报告结构组织
   - 配图和链接
   - 审核发布
```

## 工具组合

```
RSS监控 → 内容采集 → 去重过滤 → 阅读摘要 → 聚合整理 → 发布输出

[ rss-monitor ] → [ content-harvester ] → [ content-summarizer ] → [ content-aggregator ] → [ content-publisher ]
```

## 注意事项

- 每条内容必须标注来源（可追溯）
- 同一事件多报道时选权威/一手来源
- 负面/批评内容同样客观呈现
- 简报控制在约定篇幅内
- 生成后自己先读一遍再发出
