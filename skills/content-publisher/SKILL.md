### 5. Telegram 频道

## 发布前检查清单

- [ ] 内容已通过摘要和提炼
- [ ] 标题符合平台规范（Twitter ≤ 280字符）
- [ ] 包含原始链接/引用来源
- [ ] 标签/话题已添加（#标签）
- [ ] @提及和链接已验证
- [ ] 图片/附件已准备

## 各平台适配规则

### Twitter/X
- 标题：抓眼球，有悬念或数据
- 正文：开门见山，140字内最佳
- 末尾：加原始链接/话题标签
- 配图：16:9 最佳，2张以下

### 微信公众号
- 标题：情绪化，可带感叹号
- 开头：制造紧迫感或共鸣
- 结构：分段清晰，3-5图配文
- 末尾：引导转发/点赞

### 飞书文档
- 结构：多级标题，目录清晰
- 内容：偏工作向，知识性强
- 协作：适合团队共享

### 博客
- SEO：关键词在标题和首段
- 格式：Markdown，图片 alt 标签
- 分类：选对分类和标签
- 更新：主动提交搜索引擎

## 发布工作流

```
1. 选择目标平台
2. 内容适配（长度/风格/格式）
3. 预览确认
4. 执行发布（或存入草稿）
5. 记录发布状态（更新Obsidian）
6. 监控反馈（评论/转发/阅读）
```

## 注意事项

- 敏感内容发布前必须确认
- 保持各平台人设一致性
- 重要内容先发草稿自己审阅
- 发布后更新原始文档的 status 字段
