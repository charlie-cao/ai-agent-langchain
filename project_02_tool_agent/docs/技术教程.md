## 二、核心概念：ReAct 是什么？

**ReAct = Reason + Act**

这是一种让 LLM "思考-行动-观察" 的循环模式：

```
Think: "我需要知道 15+27 的结果"
Action: call calculator tool with "15 + 27"
Observation: "42"
Think: "结果拿到了，可以回答用户了"
Action: Final Answer: "42"
```

## 三、代码逐行解析

### 3.1 agent.py — 主入口

#### 第一部分：工具定义

```python
# All available tools
ALL_TOOLS = [web_search, calculator, file_read, file_write, file_list, get_datetime]
```

**讲解**：这就是 Agent 的"武器库"。每个工具都是独立的函数，LLM 可以自由选择调用哪个。

---

#### 第二部分：LLM 工厂模式

```python
def _llm(streaming: bool = False) -> ChatOllama:
    return ChatOllama(
        model=DEFAULT_MODEL,        # "qwen3.5:latest"
        base_url=OLLAMA_BASE_URL,  # "http://localhost:11434"
        temperature=TEMPERATURE,   # 0.1 (低温度=更确定性回答)
        streaming=streaming,
    )
```

**讲解**：
- 使用**工厂模式**创建 LLM 实例
- `ChatOllama` 来自 `langchain_ollama` 包（新版）
- `streaming` 参数控制是否流式输出

---

#### 第三部分：全局 Agent 缓存

```python
_agent = None

def get_agent():
    global _agent
    if _agent is None:
        _agent = create_react_agent(
            model=_llm(),
            tools=ALL_TOOLS,
        )
        logger.info("Agent initialized")
    return _agent
```

**讲解**：
- **单例模式**：整个进程只创建一次 Agent
- `create_react_agent` 是 LangGraph 提供的预建 Agent
- 它内部已经处理好了 ReAct 循环逻辑

---

#### 第四部分：核心运行函数

```python
def run(question: str, chat_history=None) -> dict:
    t0 = time.perf_counter()
    
    agent = get_agent()
    
    # 构建消息列表
    messages = []
    if chat_history:
        messages.extend(chat_history)
    messages.append(HumanMessage(content=question))
    
    # 调用 Agent
    result = agent.invoke({"messages": messages}, stream_mode="values")
```

**讲解**：
- `HumanMessage` 是 LangChain 的消息类型，代表用户输入
- `agent.invoke()` 是同步调用，会等待 Agent 完成
- `stream_mode="values"` 返回完整的消息历史

---

#### 第五部分：从结果中提取工具调用

```python
    steps = []
    result_messages = result.get("messages", [])
    for msg in result_messages:
        if hasattr(msg, "type") and msg.type == "tool":
```

**讲解**：

### 3.2 calculator_tool.py — 安全计算器

#### 核心：AST 安全计算

```python
_ALLOWED_OPS = {
    ast.Add: operator.add,    # +
    ast.Sub: operator.sub,    # -
    ast.Mult: operator.mul,    # *
    ast.Div: operator.truediv, # /
    ast.Pow: operator.pow,    # **
    ast.USub: operator.neg,   # 负号
    ast.Mod: operator.mod,    # %
    ast.FloorDiv: operator.floordiv,  # //
}
```

**为什么用 AST 而不是 eval()？**

| 方法 | 安全性 | 原理 |
|------|--------|------|
| `eval("2+2")` | ❌ 危险 | 执行任意 Python 代码 |
| AST 解析 | ✅ 安全 | 只解析数学表达式，不执行代码 |

```python
def _safe_eval(node):
    if isinstance(node, ast.Constant):
        return node.value  # 直接返回值
    if isinstance(node, ast.BinOp):
        left = _safe_eval(node.left)
        right = _safe_eval(node.right)
        return _ALLOWED_OPS[type(node.op)](left, right)
```

**讲解**：
- `ast.parse("2+3", mode="eval")` 把字符串解析成 AST 语法树
- `_safe_eval` 递归遍历语法树，只做数学运算
- 即使输入 `"__import__('os').system('rm -rf')"`，也只会报语法错误

### 3.3 file_tool.py — 沙箱文件操作

#### 核心：路径穿越防护

```python
SANDBOX = Path(__file__).parent.parent / "workspace"

def _safe_path(filename: str) -> Path:
    p = (SANDBOX / filename).resolve()
    if not str(p).startswith(str(SANDBOX.resolve())):
        raise ValueError(f"Path traversal attempt blocked: {filename!r}")
    return p
```

**讲解**：
- 所有文件操作**只能在 `./workspace/` 目录下**
- `resolve()` 把相对路径变成绝对路径
- 检查最终路径是否以 `SANDBOX` 开头

**攻击示例**：

```python
# 恶意输入: "../../../etc/passwd"
# 攻击者想读取系统密码文件

# _safe_path 会这样处理：
# SANDBOX = C:\work\project\workspace
# 输入   = ../../../etc/passwd
# 解析后 = C:\work\project\workspace\..\..\..\etc\passwd
#         = C:\etc\passwd  ← 不在 SANDBOX 下！
# 防护机制: 检测到路径越界，直接拒绝！
```

### 3.4 datetime_tool.py — 时区时间

```python
from zoneinfo import ZoneInfo  # Python 3.9+ 内置

@tool
def get_datetime(timezone_name: str = "UTC") -> str:
    tz = ZoneInfo(timezone_name)  # 支持 IANA 时区名
    now = datetime.now(tz)
    return now.strftime("%Y-%m-%d %H:%M:%S %Z")
```

**讲解**：
- 使用 `zoneinfo` 而不是 `pytz`（Python 3.9+ 内置）
- 支持标准 IANA 时区名：`"UTC"`, `"Asia/Shanghai"`, `"America/New_York"`

### 3.5 search_tool.py — 网页搜索

```python
@tool(parse_docstring=True)
def web_search(query: str) -> str:
    """
    Use this tool when you need to:
    - Find current events, news, or prices
    - Look up facts or documentation
    
    Args:
        query: The search query. Be specific.
    """
    search = DuckDuckGoSearchRun(max_results=MAX_SEARCH_RESULTS)
    return search.run(query)
```

**讲解**：
- `@tool` 是 LangChain 的装饰器，自动处理参数解析
- `parse_docstring=True` 让 LLM 能读懂工具的文档
- `DuckDuckGoSearchRun` 来自 `langchain-community`
- **不需要 API Key**，直接调用 DuckDuckGo

## 七、常见问题解答

**Q: 为什么用 LangGraph 而不是 LangChain Agent？**

A: LangGraph 是更现代的架构，支持状态管理、循环控制、可持久化。LangChain Agent 已deprecated。

**Q: Agent 是怎么知道该调用哪个工具的？**

A: LLM 会读工具的 docstring，结合用户问题，决定是否调用、调用哪个。

**Q: 流式输出是怎么实现的？**

A: `stream_mode="updates"` 会把每一步的结果实时 yield 出来。

**Q: 搜索功能需要 API Key 吗？**

A: 不需要！使用 DuckDuckGo，无需注册即可搜索。

**Q: 如何添加新工具？**

A: 只需定义一个带 `@tool` 装饰器的函数，然后在 `ALL_TOOLS` 列表中添加即可。
